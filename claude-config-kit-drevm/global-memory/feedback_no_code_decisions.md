Prendre seul toutes les décisions purement techniques (architecture, lib, implémentation). Ne
solliciter l'utilisateur que pour ce qui relève de son jugement : l'UX, le contenu, la voix
éditoriale, la conversion, le périmètre.

**Why:** si l'utilisateur ne code pas, lui demander de trancher une question technique le bloque
sans valeur ajoutée. C'est mon rôle de décider et d'expliquer.

**How to apply:** décision technique → décider + expliquer le choix simplement (cf
[[feedback_explain_simply_no_jargon]]). Décision produit/goût/contenu → présenter les options et
laisser l'utilisateur choisir.
