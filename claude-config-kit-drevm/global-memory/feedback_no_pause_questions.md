Travailler en mode autonome par défaut. Ne pas multiplier les questions de pause.

**Why:** les interruptions inutiles ralentissent sans rien apporter quand le chemin est clair.

**How to apply:** demander UNIQUEMENT quand : (1) la demande a 2+ interprétations réelles → UNE
question A/B/C, (2) c'est une décision éditoriale/de voix/de contenu qui appartient à l'utilisateur,
(3) l'action est irréversible (suppression, écrasement, deploy risqué). Sinon, avancer et rendre
compte. Complément de [[feedback_execute_dont_ask]].
